Plan: Gangsheet PDF Service

Dịch vụ in DTF: User chỉ upload ảnh → nest gang-sheet → save/submit đơn. Đội nội bộ (edit, ops, in, vận chuyển) xử lý toàn bộ pipeline sau submit — user không thấy sửa file, render PDF hay in ấn.

Greenfield · User app + Ops pipeline · PNG / WEBP / AVIF · ảnh 20–100 MB

Checklist triển khai

Hai thế giới: User vs Nội bộ

User (khách hàng)

Upload ảnh → Build gang-sheet → Save / Submit đơn → theo dõi trạng thái: Đã gửi → Đang xử lý → Đang in → Đã giao.

Không thấy: edit file, replace ảnh, render PDF, cảnh báo DPI kỹ thuật.

Nội bộ (edit team + ops + in ấn)

Nhận đơn SUBMITTED → edit team sửa file trên server → ops refresh & render PDF → in DTF → vận chuyển.

Replace/refresh ảnh và Generate PDF chỉ trên backoffice / ops tool.

Bối cảnh & vai trò

In DTF trên film rộng 22 inch. File thiết kế thường 20–100 MB. User thiết kế layout; xưởng đảm nhận sửa ảnh, xuất PDF, in và giao hàng.

Triển khai theo phase

Phase 1 — MVP

  • User FE: upload → nest → submit order
  • BE: images + orders + snapshot + ops render PDF
  • Ops backoffice tối thiểu: list đơn, render PDF, đổi status
  • SOP edit team (file server)
  • PNG / WEBP / AVIF, ảnh lớn

Phase 2

  • Auth user vs ops roles
  • Job queue auto-render sau khi ops mark edit done
  • Webhook / manual SHIPPED

Phase 3

  • S3 storage, async render, multi-tenant

Definition of Done (MVP)

  • User submit đơn sau nest — không thấy bước edit/render
  • Edit team ghi đè file; ops render PDF từ snapshot + file mới
  • PDF đúng layout đã submit; ảnh 20 MB+ OK
  • User xem được status SUBMITTED → SHIPPED
  • SOP + README ops/edit team

Plan: Gangsheet PDF Service

Dịch vụ in DTF kiểu Ninja Transfers: User upload ảnh → nest trên canvas 22" → Save draft hoặc Save & Add to Cart → checkout. Ảnh lưu trên S3 (URL trong snapshot). Không có flow đội edit ghi đè file trên server.

Spring Boot 3 · H2 (dev) → PostgreSQL · Gang sheet draft/confirmed · Cart · Order

Checklist triển khai

Hai thế giới: User vs Nội bộ

User (khách hàng)

Upload ảnh (S3) → Build / nest gang-sheet trên canvas → Save hoặc Save & Add to Cart → checkout.

Có thể tiếp tục sửa design khi status = draft (kể cả đã có trong cart). Chỉ khóa khi checkout.

Không thấy: render PDF, pipeline in ấn nội bộ.

Nội bộ (ops + in ấn)

Nhận đơn SUBMITTED → render PDF từ snapshot đã lưu (ảnh S3 URL) → in DTF → vận chuyển.

Không có bước edit team sửa file trên server.

Các bước trên builder

BướcUI (Ninja)API
1. Upload Drag-drop ảnh · gallery bên trái FE → S3 (URL dùng trong items[].img)
2. Build Canvas 22" · auto-nest · margin · kéo/xoay GET /api/gang-sheets/config · optional POST /api/nesting
3a. Save Nút Save — lưu để dùng sau POST /api/gang-sheets
3b. Save & Add to Cart Nút Save & Add to Cart (qty mặc định 1) POST /api/gang-sheets/cart · đổi qty: PATCH /api/cart/items
4. Checkout Thanh toán / đặt hàng POST /api/orders

Save & Add to Cart

POST /api/gang-sheets/cart
{
  "design_id": "uuid",
  "name": "...",
  "sheet": { ... },
  "items": [ ... ]
}

→ MGS_42 — in_cart: true, cart_quantity: 1 (mặc định)

quantity optional — FE thường không gửi. Lần đầu add = 1 tấm. Re-save từ cart edit (không gửi quantity) giữ qty hiện tại. Đổi số lượng: PATCH /api/cart/items/{line_id}.

Layout: Builder vs Print

Giai đoạnAi tính sheet heightQR padding
Save draft (builder)FE gửi sheet.height_inch sau nestKhông
Gen PDF (print)BE computePrintLayoutCó — có thể cao hơn builder

User canvas khớp builder height. File in có thể dài hơn để chừa QR — items không đổi vị trí.
